# iOS TestFlight Deployment via Local Xcode Build

## Overview

Deploy "The Executive" to TestFlight using `expo prebuild` to generate the native Xcode project, then archive and upload via Xcode. No EAS or third-party services needed — just Xcode and an Apple Developer account.

## Prerequisites

- Apple Developer Program enrollment ($99/yr) with certificates/provisioning ready
- Xcode installed on macOS
- Bundle ID `com.executive.app` registered in Apple Developer portal
- An app record created in App Store Connect (required for TestFlight uploads)

## Build Flow

```
npx expo prebuild --platform ios --clean
       ↓
  ios/ directory generated (Xcode workspace + CocoaPods)
       ↓
  Open ios/TheExecutive.xcworkspace in Xcode
       ↓
  Set signing team → Automatically manage signing
       ↓
  Product → Archive
       ↓
  Xcode Organizer → Distribute App → TestFlight
```

## app.json

Current config is sufficient. Key fields:

```json
{
  "expo": {
    "name": "The Executive",
    "slug": "the-executive",
    "version": "1.0.0",
    "ios": {
      "supportsTablet": true,
      "bundleIdentifier": "com.executive.app"
    }
  }
}
```

The `bundleIdentifier` must match the App ID registered in the Apple Developer portal.

## package.json Scripts

Add convenience scripts:

```json
"prebuild:ios": "npx expo prebuild --platform ios --clean"
```

The `prebuild:ios` script regenerates the native `ios/` directory from scratch. The `--clean` flag removes any previous native project first, ensuring a fresh build.

Archiving and uploading are done through Xcode GUI (Product → Archive → Distribute), not via CLI scripts.

## .gitignore

Already configured correctly — `/ios` and `/android` are gitignored since they're regenerated by `expo prebuild`.

## Signing

Xcode handles signing automatically when:
1. You select your Apple Developer team in Signing & Capabilities
2. "Automatically manage signing" is enabled

Expo prebuild sets the bundle identifier from `app.json`, so Xcode matches it to your provisioning profile automatically.

## Deployment Steps (Manual)

1. Bump `version` in `app.json` if needed
2. Run `npm run prebuild:ios`
3. Open `ios/TheExecutive.xcworkspace` in Xcode
4. Select your signing team under Signing & Capabilities
5. Select "Any iOS Device (arm64)" as the build target
6. Product → Archive
7. In Organizer, click Distribute App → TestFlight & App Store → Upload
8. Wait for App Store Connect processing (~15-30 min)
9. Enable the build for TestFlight testing in App Store Connect

## Out of Scope

- Automated CI/CD (GitHub Actions, Fastlane)
- App Store production submission
- OTA updates (expo-updates)
- Push notification certificates
- App Store metadata (screenshots, description)
