# i18n with i18next — The Executive

## Overview

Add internationalization to The Executive (Eisenhower Matrix task manager) using i18next, react-i18next, and i18next-cli. Supports English (base), Spanish, French, and Russian. Device locale auto-detection with manual override in Settings.

## Stack

| Package | Purpose |
|---------|---------|
| `i18next` | Core translation runtime |
| `react-i18next` | React bindings (`useTranslation` hook) |
| `expo-localization` | Detect device locale |
| `i18next-cli` | CLI extraction tool (`npm run i18n:extract`) |

## Key Decisions

- **Natural language keys**: English text IS the key — `t('What needs to be done?')`, not `t('add.placeholder.title')`
- **Flat JSON per locale**: `src/locales/{en,es,fr,ru}.json` — no namespacing, single file per language
- **All locales loaded upfront**: ~160 keys per language, trivial bundle size — no lazy loading needed
- **No RTL support**: Out of scope for now

## Translation File Format

### `src/locales/en.json`
```json
{
  "Focus": "Focus",
  "Tasks": "Tasks",
  "What needs to be done?": "What needs to be done?",
  "{{count}} active task_one": "{{count}} active task",
  "{{count}} active task_other": "{{count}} active tasks"
}
```

### `src/locales/es.json`
```json
{
  "Focus": "Enfoque",
  "Tasks": "Tareas",
  "What needs to be done?": "¿Qué necesitas hacer?",
  "{{count}} active task_one": "{{count}} tarea activa",
  "{{count}} active task_other": "{{count}} tareas activas"
}
```

### Russian plural forms

Russian has 3 plural forms (one, few, many). i18next handles this natively:

```json
{
  "{{count}} active task_one": "{{count}} активная задача",
  "{{count}} active task_few": "{{count}} активные задачи",
  "{{count}} active task_many": "{{count}} активных задач"
}
```

## Initialization

### New file: `src/lib/i18n.ts`

- Initialize i18next with `react-i18next`'s `initReactI18next` plugin
- Import all 4 locale JSON files as resources
- Detect device locale via `expo-localization` (`getLocales()[0].languageCode`)
- Fall back to `en` if detected locale is unsupported
- Check AsyncStorage (`@executive_lang`) for manual override — if set, use that instead of device locale
- Interpolation: `escapeValue: false` (React already escapes)

### Integration point: `app/_layout.tsx`

- Side-effect import: `import '../src/lib/i18n'` at the top of root layout
- No wrapping provider needed — `react-i18next` uses i18next's singleton

## Component Usage Pattern

### Basic translation
```tsx
import { useTranslation } from 'react-i18next';

function MyScreen() {
  const { t } = useTranslation();
  return <Text>{t('What needs to be done?')}</Text>;
}
```

### Interpolation
```tsx
t('{{count}} active task', { count: 5 })
// English: "5 active tasks"
// Russian: "5 активных задач"
```

### Quadrant definitions

The quadrant metadata in `src/types/task.ts` (title, priority, description) will be converted to translation keys. The `QUADRANTS` object will store English strings as keys, and `t()` is called at render time — not at definition time — so translations resolve correctly.

## Settings Integration

New "Language" row in `app/(tabs)/system.tsx` under the "Appearance" section (below Dark Mode):

- Label: "Language"
- Picker/selector with options: English, Español, Français, Русский
- On change: calls `i18next.changeLanguage(lng)` and persists to AsyncStorage (`@executive_lang`)
- Current language shown as the selected value

## Extraction Script

### Config: `i18next-scanner.config.js` (project root)

```js
module.exports = {
  locales: ['en', 'es', 'fr', 'ru'],
  output: 'src/locales/$LOCALE.json',
  input: ['app/**/*.tsx', 'src/**/*.tsx', 'src/**/*.ts'],
  sort: true,
  keepRemoved: false,
  keySeparator: false,
  namespaceSeparator: false,
};
```

Key config notes:
- `keySeparator: false` and `namespaceSeparator: false` — required for natural language keys (prevents `.` and `:` from being treated as separators)
- `keepRemoved: false` — removes keys from JSON that no longer appear in source
- `sort: true` — alphabetical keys for clean diffs

### npm script

```json
{
  "scripts": {
    "i18n:extract": "i18next 'app/**/*.tsx' 'src/**/*.tsx' 'src/**/*.ts'"
  }
}
```

## What's NOT Included

- **RTL layout support** — no Arabic/Hebrew for now
- **Server-side translation** — Supabase auth/error messages stay in English
- **Lazy-loading locales** — unnecessary at this scale
- **Locale-specific date formatting** — already handled by `toLocaleString()` in calendar code
- **Translation management platform** — manual JSON editing for now
- **Automated CI extraction** — manual `npm run i18n:extract` only
